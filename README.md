## Features

- **8-bit sprite characters** тАФ Each session gets a unique pixel-art character with idle/active/jump animations
- **Status bar indicators** тАФ Green blinking bar (active), red blinking bar + "NEEDS HUMAN INTERVENTION" (permission), subtle bar (idle)
- **Pixel badge project names** тАФ Retro game-style colored badges make project identity unmissable
- **Live session tracking** тАФ See all running Claude Code sessions at a glance
- **CPU-based inference** тАФ Detects active sessions even without hooks via process scanning
- **Anti-flicker** тАФ CPU hysteresis + time-based status hold prevents status jitter
- **macOS notifications** тАФ Get notified on permission requests, completions, and status changes
- **Detail panel** тАФ Drill into any session for full context: tool in use, last prompt, branch, CPU, elapsed time
- **Chat history dropdown** тАФ View conversation history inline per session
- **Hook + process fusion** тАФ Combines hook events with live `ps`/`lsof` scanning for maximum accuracy
- **UUID-based matching** тАФ Precise session-to-process linking, no ghost sessions

## Quick Start

**1. Install the Claude Code hooks:**

```bash
uv run monitorator install
```

This registers event hooks in `~/.claude/settings.json` that write session state to `~/.monitorator/sessions/`.

**2. Launch the dashboard:**

```bash
uv run monitorator
```

**3. Start using Claude Code normally.** Sessions appear automatically in the dashboard.

## CLI Commands

| Command                            | Description                                  |
| ---------------------------------- | -------------------------------------------- |
| `monitorator` or `monitorator run` | Launch the TUI dashboard                     |
| `monitorator status`               | Quick CLI status (no TUI)                    |
| `monitorator install`              | Install hooks into `~/.claude/settings.json` |
| `monitorator uninstall --clean`    | Remove hooks and clean session data          |

## How It Works

```
Claude Code hooks тФАтФАтЖТ ~/.monitorator/sessions/*.json тФАтФАтФР
                                                        тФЬтФАтФАтЖТ SessionMerger тФАтФАтЖТ TUI
ps + lsof scanning тФАтФАтЖТ ProcessInfo[] тФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФШ
```

1. **Hook** (`hooks/zig/`) тАФ A compiled Zig binary that processes Claude Code events and writes session state as JSON. ~20x faster than the Python fallback (~2.7ms vs ~53ms per invocation). Falls back to `hooks/emit_event.py` (stdlib-only Python) if the binary is not available.

2. **ProcessScanner** тАФ Finds running Claude Code processes via `ps`, resolves their working directory and session UUID via `lsof`.

3. **SessionMerger** тАФ Joins hook data with process data by matching `cwd`. Applies CPU-based status override (idle тЖТ thinking if CPU > 10%) with hysteresis to prevent flicker.

4. **TUI** тАФ Built with [Textual](https://textual.textualize.io/). Polls every 2s with diff-based updates. Each session gets a unique 8-bit sprite character, pixel badge project name, status bar indicator, and animated transitions.

### Requirements

- Python 3.11+
- macOS (process scanning uses `ps`/`lsof`; notifications use `osascript`)
- [uv](https://docs.astral.sh/uv/) for dependency management

### Hook Binary

A pre-built Zig binary is included at `hooks/zig/zig-out/bin/emit_event` for convenience.

> **Security note:** This binary was compiled on a contributor's machine. If you prefer not to trust pre-built binaries, build it yourself:
>
> ```bash
> # Requires Zig 0.15+ (https://ziglang.org/download/)
> cd hooks/zig
> zig build -Doptimize=ReleaseFast
> ```
>
> Then re-run `uv run monitorator install` to register the locally-built binary.

The installer automatically prefers the Zig binary and falls back to the Python script (`hooks/emit_event.py`) if it's not found.

### Key Conventions

- `from __future__ import annotations` in every module
- `hooks/emit_event.py` must remain stdlib-only (Python fallback)
- Tests use `pytest` with `pytest-asyncio`
